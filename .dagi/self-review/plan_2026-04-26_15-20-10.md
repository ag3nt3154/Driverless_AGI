# Plan — Session 2026-04-26_15-20-10: Shortcomings and Improvement Plan

## Context

This session attempted to run `memory-ingest` but was stopped by a "Wiki Not Initialised" warning. The actual failure cause was a misconfigured system prompt that set the project root to `G:\My Drive\black_grimoire\dagi-memory\raw` — a path inside the Google Drive folder — when the real project root is `C:\Users\alexr\Driverless_AGI`. This caused `find`, `read`, and `bash` tool failures. The agent correctly stopped instead of forcing ahead.

## Findings

### Shortcomings

1. **System prompt project root misconfiguration.** The system prompt for this session contained `Project root: G:\My Drive\black_grimoire\dagi-memory\raw`, which is inside the `raw/` folder itself. The real project root is `C:\Users\alexr\Driverless_AGI`. This is not an agent error — it is a session configuration error.

2. **Path guard too restrictive for `G:` drive.** The path guard tool allows reads from `G:\My Drive\black_grimoire\dagi-memory\raw` (the session's configured root) but blocks `G:\My Drive\black_grimoire\dagi-memory\wiki` even though both directories are on the same drive. When the agent tried to use an absolute `G:` path for the log file, the path guard rejected it.

3. **`bash` path inconsistency on Windows.** The `ls` command in bash failed to list the `G:` directory while `dir` in the same bash tool succeeded. The skill should recommend `dir` over `ls` for Windows path operations.

4. **No automatic bash fallback.** When `read`/`find`/`write` failed on `dagi-memory/` paths, the agent did not automatically try `bash` as a fallback — it only tried `bash` after explicit failure of the other tools. The `memory-ingest` skill should include guidance to always use `bash` for `G:` path operations.

### Areas of Improvement

1. **Session configuration hygiene.** The system prompt project root should be validated at session start to prevent tools from resolving to incorrect paths.
2. **Path guard scope.** The path guard should cover the full `dagi-memory/` directory tree on `G:`, not just the `raw/` subfolder.
3. **Skill tool guidance.** `memory-ingest` should instruct agents to use `bash` with absolute `G:` paths for all file I/O from the first step, rather than expecting `read`/`find`/`write` to work.
4. **Windows bash conventions.** Skills should recommend `dir` rather than `ls` for Windows path operations.

### Improvement Items

1. `[high] Validate project root in system prompt` — add a startup check that warns if the system prompt's project root does not match the actual filesystem path
2. `[high] Extend path guard to cover full dagi-memory tree on G:` — allow `wiki/` and `sources/` subdirectories in path guard roots
3. `[medium] Add bash-fallback guidance to memory-ingest` — all dagi-memory file operations should use bash with absolute paths on G:
4. `[low] Recommend `dir` not `ls` in memory skills` — Windows bash sessions should use `dir` for directory listing on G:
